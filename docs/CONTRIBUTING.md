# Kola UI – Contribution Guide

## How to Contribute

- Fork the repository or clone locally.
- Create a new branch for your feature or fix.
- Follow the code style of existing files.
- Add tests or usage examples if possible.
- Document new features in README/docs.
- Submit a pull request or patch.

## Code Style

- Use clear, descriptive class and method names.
- Keep components small and focused.
- Prefer composition over inheritance where possible.
- Document public APIs with Javadoc.

## Adding Components

- Place new components in `src/components/`.
- Update `module-info.java` if needed.

## Extending Styles

- Add new rules in `style/TailwindParser.java`.

## Reporting Issues

- Open an issue with steps to reproduce and expected behavior.

---

Thank you for contributing!
